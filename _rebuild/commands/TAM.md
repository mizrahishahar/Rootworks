# TAM

## When to do it
Before a build, or whenever a niche's size is in question: is it big enough for a cold email test?

## What we get
A TAM brief: a reachable company count and a decision-maker headcount with the math shown, a viability read, the target buyer titles, and the load-bearing assumptions.

## Process

### 1. Confirm the niche
**Owner:** Claude
Confirm it is specific enough to estimate: subsector, geography, and size band. Ask only if ambiguity would swing the estimate; if it is already specific, proceed.

### 2. Estimate
**Owner:** Claude · **Skill:** tam-estimator
Run the estimate and produce the brief.

### 3. Read the verdict
**Owner:** Operator
Under ~1,000 reachable, do not build. Under 10,000, note the exhaustion risk. Above that, the niche clears.
