# Output Interface

Two artifacts. Shapes are fixed.

## 1. Lead Card

Shown in the working conversation before the message, one per lead, same field order every time.

- Company (heading) and Email
- Contact: name, role, DM status (confirmed / gateway / unknown)
- Location, size, ICP score
- Timezone and the prospect's current local hour
- Persona (the sending inbox) and the thread subject
- Qualification brief: verbatim from the lead's stored qualification
- Thread: chronological, the latest inbound reply marked
- Read: the named use case (from The Playbook) and the one-line judgment that sets the move

## 2. Message

The drafted email: in the persona's voice, no invented duration, no calendar link, signed as the sending persona. The Lead Card precedes it; asked for standalone, the message is the whole output.

---

Note: there is no per-reply log and no client knowledge note here. Durable company-specific grounding lives in the Company Overrides file; per-run narrative, when a batch run is run, is the batch SOP's concern, not the skill's output. Continuity comes from reading the full thread every time, not from a stored log.
