# Company Overrides Interface

A slim, concise file that overrides the general skill for one prospect. Nothing more. It is the short list of instructions that change how the general inbox manager behaves for this specific company, the things it could not know, or would get wrong, without being told.

It is not a knowledge base, not a CRM, not a persona list, not an objection script. Company facts live in the onboarding form. Assets live in the Assets file. Lead state lives in PlusVibe. This file holds only the overrides.

Location: Prospects/{Client}/Assets/INBOX-MANAGEMENT/Company Overrides.md

A short list, each item a specific instruction that bends the general skill for this prospect: a feature to lead with that the skill would otherwise avoid, a tool or channel quirk to work around, a verification gap not to trust, a hard do or do-not unique to this company. If there is little to override, the file is nearly empty, and that is correct.

**Precedence:** overrides bend emphasis and facts (what to lead with, what to avoid, tool quirks). They never replace the Playbook framework: outcome leads, the use-case structure holds, an override rides inside it.
