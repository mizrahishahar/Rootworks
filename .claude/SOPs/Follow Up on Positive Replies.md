## When to do it
Cold-email replies are sitting in the interested, no-show, or booked-not-yet-shown buckets and need working as a batch.

## What we get
Every interested-and-not-booked and no-showed lead moved one step toward a booked call, every booked-not-yet-shown lead protected toward showing up, the prospect's Company Overrides updated with anything durable the run surfaced, and any general craft lesson folded back into the skill.

## Process

### 1. Ready up on the skill
**Owner:** Claude · **Skill:** inbox-manager

Invoke the inbox-manager skill and read it in full — the skill and its entire knowledge base, end to end, plus its interfaces. This is a dedicated phase, not a warm-up: understand it deeply, understand the dedication that went into it, and follow it through the whole run. Do not skim, do not work from memory of a prior run; the whole skill is read fresh every time. Not a single lead is touched before this is done.

### 2. Load the client
**Owner:** Claude

Load this client's full inbox-management setup end to end: the onboarding form, the Company Overrides, and the Assets file — now, with the skill fresh, understanding exactly what bends on top of it for this client. Read the Company Overrides closely and honor every bend they declare, especially any that override a default this run would otherwise assume (where the stored qualification lives, the booking tool and calendar, the language, what to lead with or avoid). Then establish the booking availability source: if a calendar connector is wired, verify it responds; if not, get the working window's availability from the Operator now, once, up front. If the lead source (PlusVibe) errors at any point in the run, do not continue and do not work around it: flag it to the Operator, wait for the fix, then pull live.

### 3. Correct the labels
**Owner:** Operator

Make the buckets the source of truth. Interested holds only interested-and-not-booked leads; booked moves to meeting booked; completed to meeting completed; no-shows carry no-show. The sweep works whatever the labels say, so a dirty label works the wrong lead.

### 4. Pull the worklist and work it
**Owner:** Claude · **Skill:** inbox-manager

Pull all three label buckets live: interested, no-show, and booked-not-shown. Present the worklist as a clear, formatted table: each bucket, how many leads in it, the leads by name, and which one we start with and why. Agree scope with the Operator, then work it most-relevant-first, one lead at a time, with a gate between each.

For each lead: pull the full thread, present the Lead Card per the skill's interface, and draft the one move the situation calls for.

### 5. Approve and send
**Owner:** Operator

Review the Lead Card and the draft, refine, approve, send. Then return to step 4 for the next lead. Repeat until the worklist is empty.

### 6. Close the run
**Owner:** Claude · **Skill:** inbox-manager

Promote anything durable the run surfaced into this prospect's Company Overrides (a confirmed objection answer, a new persona, a lead-with or avoid call), keeping the file tight and cutting anything it made stale. If the run surfaced a general craft lesson about how the inbox manager itself should work, run improve-skill to fold it into the skill. Leave operational carry-forward (suppressions, agent fixes) in the run summary to the Operator, not in any per-client file. Then the run is closed.
