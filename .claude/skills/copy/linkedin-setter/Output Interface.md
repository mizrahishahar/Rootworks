# Output Interface

Three artifacts, one per prospect, same field order every time. Shown in the working conversation. The shapes are fixed.

## 1. Lead Card

A snapshot of who they are and where the thread stands.

- **Name** (heading), title, company
- **LinkedIn URL**
- **Airtable**: status plus record link, read-only. Or "not in CRM yet". Never written to.
- **ICP / offer-fit**: verbatim from the record's qualification or contextNotes, if it exists
- **Thread state**: first-touch, or follow-up #N with what the last touch was
- **Goal**: earn a reply from the right person

## 2. Fresh Research

The live dig, run every time before drafting. This is where the effort shows.

- **Recent LinkedIn posts / activity** (last ~30 to 60 days): what they are lit up about
- **Signals**: funding, hiring, launches, news
- **Their positioning in their own words**
- **The hook**: the one specific, true thing this message anchors on

## 3. The Move

- **The read** (one line): where the thread is, what this message must do, the value angle
- **Deploying**: which implemented skill writes the words (cold-offer-generator, cold-email-copywriter, sales-closer), or "drafting directly"
- **The message**: first DM or the specific follow-up. Hebrew for Israeli founders, in a blockquote. Value-anchored and hyper-personalized, never a generic pitch.
- **English gloss**
- **Handoff**: any reply, positive or negative, hands the thread to the inbox-manager

---

Note: no CRM write and no stored log live here. The thread is pasted by the operator each run; continuity comes from reading the full thread every time, not from a stored record. The skill stops at the first reply.
